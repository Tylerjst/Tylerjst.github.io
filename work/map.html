Emory's Sustainability Map
Renovating OSI's Technology

The Office of Sustainability Iniatives' website featured an outdated sustainability map. It was my job to renovate it.

Explore the map on your own.

I. Map 1.0

The intention of OSI's sustainability map is to centralize geographic information about Emory's sustainability features. Things like LEED certified buildings, bike share locations, or CLIFF shuttle stops could be rendered onto an easy to read map that users could use to wayfind and learn more about Emory's sustainability initiatives.

Unfortunately, the first version of this map wasn't super user-friendly. It used a flat, vector map of Emory created in Illustrator as the base layer. The additional features could be toggled by "adding" that flat image layer on top of the base layer.

Check out the map for yourself here

II. Map 2.0

As OSI's intern that summer, I quickly updated the old map with updated shape layers then got started on creating a newer map. Fortunately, OSI had been planning on creating a new map for a while. I worked with staff in the Center for Digital Scholarship, where they had been collecting GeoJSON data from the University to implement with Leaflet, a javascript framework used for building interactive maps.

Although I had never used Leaflet before, I was confident I could familiarize myself with the framework. I got started building the new map.

The new map was the first legitimate web app that I ever built, so learning how to use the framework was definitely a challenge. I did have some staff available in the ECDS to give some pointers on how to access the GeoJSON data and connect to Emory's severs via AJAX, but for the most part, I was on my own for the project.

III. A New Hope

To counteract the pressure, I took the whole endeavour as one big trial-and-error experiment. I posted the existing layers from the old map (stuff like Garden Market and ZipCar locations), but also tried playing with new ideas, like MARTA bus paths and Emergency Phone Locations. This GeoJSON information was publically available.

What I found was that the Leaflet framework displayed this information way more effectively than Map 1.0. Not only was the base map easier to read and navigate, but the location markers were much nicer to look at and could display additional information about that waypoint in the bottom right corner. I also tried tying in my UX and graphic design skills by toying with custom markers to differentiate the features and develop OSI's own "flair" to the map.

However, I ran into quite a number of problems along the way. I posted a good amount of features onto Map 2.0, but I found myself waiting for a huge amount of addtional data from other campus partners. For example, I was unable to display Cliff Shuttle bus routes and stops because I was never given access to that GeoJSON data from Parking and Transportation Services (I promise I asked nicely).

By the time my internship ended, the map was still incomplete. I still wanted to update several layers on the map, as well as develop different sets of markers for the individual layers, but I simply ran out of time. This was a hefty project that was left in solely my hands. Considering this, I'm pretty proud with what I was able to learn and build in a short couple of months.

TL;DR:

  • Created new AI layers for old map.
  • Learned Leaflet Javascript framework.
  • Created new interactive JS map.
  • Initiated development of brand identity for the new map.
